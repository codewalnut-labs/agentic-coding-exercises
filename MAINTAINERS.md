# Maintainer Notes

The purpose of this repository is to preserve reusable exercises while still letting learners get realistic pull request feedback.

## PR Policy

- Review learner PRs for feedback.
- Do not merge learner solution PRs into `main`.
- Close learner PRs after review.
- Use labels such as `exercise-1`, `exercise-2`, `exercise-3`, `feedback-ready`, and `reviewed`.

## Admin-Only Changes

Only maintainers/admins should change:

- Exercise prompts.
- Starter code.
- CI and repository policy.
- Branch protection settings.
- Rubrics and expected deliverables.

## Suggested Review Style

Lead with the highest-impact finding. For code exercises, call out behavior regressions, missing tests, security risks, and production-readiness gaps before style suggestions.
